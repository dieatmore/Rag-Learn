# PDF 解析流程

## 四步流程

```
┌───────────────┐     ┌──────────────────┐     ┌──────────────┐     ┌────────────┐
│ 1. Python脚本  │ ──► │ 2. 拼接规则文档   │ ──► │ 3. AI执行拼接 │ ──► │ 4. 人工校验 │
│   机械提取     │     │   (本文件)        │     │   结构化输出   │     │   确认无误   │
└───────────────┘     └──────────────────┘     └──────────────┘     └────────────┘
     产出:                    定义:                  产出:                  产出:
  - 纯文本                  如何从碎片重建        - 结构化 Markdown       - 准确的知识库
  - 表格坐标                层级关系的规则        - 可直接分块使用
  - 布局文字
```

### 第1步：Python 脚本机械提取（`scripts/extract_pdf.py`）

**做什么**：用 pdfplumber 从 PDF 中提取所有文字和坐标，不做任何智能拼接。

**产出文件**：

| 文件 | 内容 | 用途 |
|------|------|------|
| `pdf_extract_text.txt` | 按页提取的纯文本 | 快速浏览，对照原始 PDF |
| `pdf_extract_table.json` | 表格行列数据 + cell 坐标 + 是否合并 | 按表格结构提取，带位置信息 |
| `pdf_extract_layout.json` | 按 y 坐标排序的所有文字行 | 兜底数据——当表格检测失败时使用 |

**为什么这一步不够**：PDF 表格有大量合并单元格和跨页结构，脚本只能拿到"碎片"——知道每个格子里有什么文字，但不知道格子之间的层级关系。

### 第2步：拼接规则（本文件定义）

**做什么**：定义一组规则，告诉 AI 如何从第1步的碎片中重建层级。

**规则清单**：

#### 规则1：识别一级指标边界

| 信号 | 说明 |
|------|------|
| 出现「总分 XX」格式 | 如"学术专长 50"，之后直到下一个一级指标的所有内容都属于它 |
| 新页面出现粗体标题 | 如第2页"文体特长"、第3页"其他方面" |
| 文本中出现「上限XX分」 | 如"外语能力(上限10分)" |

#### 规则2：识别二级指标归属

- 看表格中"二级指标"列（通常是第3列）是否有文本
- 该行以下、直到下一个二级指标之间的所有"等级→分值"行都属于它
- 例：`[2]=学科竞赛` 下面有第一等级50、第一等级40...直到 `[2]=高质量论文` 才切换

#### 规则3：提取等级→分值映射

- 表格中"等级"列和"分值"列直接取
- 如果等级列因合并单元格为空，继承上一行的等级
- 第3页列结构变化（7列→5列）时，根据列宽判断

#### 规则4：备注归属判断

- 备注出现的位置紧跟在某个二级指标标题之后
- 同一二级指标下所有等级共享其备注（纵向合并单元格）
- 如果备注中引用了特定指标名称（如"科技竞赛I指..."），则属于该指标

#### 规则5：跨页连续性

- 如果第 N 页末尾数据行没有标题（只有分值），查找第 N+1 页开头是否有延续
- 对照原始 PDF 确认跨页的完整标题

### 第3步：AI 按规则执行拼接

**做什么**：把第1步的粗提取产物 + 第2步的规则交给 AI，生成结构化文档。

**输入**：`pdf_extract_text.txt` + `pdf_extract_table.json` + 上述5条规则

**输出**：`pdf_structured_manual.md`

**AI 做了什么**：
1. 读粗提取文本，找到所有一级指标（学术专长/文体特长/其他方面/外语能力）
2. 逐页对照表格 JSON，为每个一级指标找下属的二级指标
3. 为每个二级指标列出所有等级→分值映射
4. 把备注文本归属到正确的二级指标下
5. 补上认定部门（从 PDF 中对应的列提取）
6. 确认跨页连接（如第1页→第2页的CSP认证延续）

### 第4步：人工校验

**要做什么**：对照原始 PDF，逐条确认 `pdf_structured_manual.md` 的正确性。

**校验清单**：

| 校验项 | 检查方式 | 重点关注 |
|--------|---------|---------|
| 一级指标是否完整 | 对照 PDF 页数，确认没有遗漏大类 | 第3页的"其他方面"是否完整 |
| 二级指标归属是否正确 | 看每个二级指标是否挂在正确的一级指标下 | 科技竞赛I/II 应在学术专长下 |
| 分值数字是否准确 | 逐个核对等级→分值数字 | 一等奖50分/二等奖40分等数字 |
| 备注归属是否正确 | 看每条备注是否挂在正确的二级指标下 | 代表作制、限项规则、日期要求 |
| 认定部门是否正确 | 核对 PDF 最后一列 | 体育部/团委/学院/党委武装部 |

---

## 为什么不用纯自动解析

这个 PDF 有三个特征让纯自动解析不可靠：

1. **纵向合并单元格**：一级指标跨20+行，自动填充会丢失归属
2. **跨页列结构变化**：第3页7列变5列，自动映射会错位
3. **备注跨行合并**：同一条备注覆盖6~8个等级行，自动复制会产生错误关联

结论：自动粗提取 + 规则 + AI拼接 + 人工校验，是这个场景下准确率最高、成本最低的方案。
